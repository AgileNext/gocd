  • In GoCD the server controls everything. It provides the ui to users, and provides work for the agents. The agents are the ones that do work (run commands, do deployments, etc).

    The server does not do any user-specified "work" on its own. It will not run any commands or do deployments. That is the reason you need a GoCD Server and at least one GoCD Agent installed before you proceed.

  • Task

    A task is something that runs on an agent, e.g. a command.

    Job

    A job is a sequential collection of tasks. All of the tasks run on a single agent's environment.

    Stage

    A stage is a nonsequential collection of independent jobs. Since the Jobs are nonsequential and independent, they can be ran in parallel.

  • A core concept in GoCD are materials. A material is a cause for a pipeline to "trigger" i.e. to start doing what it is configured to do. Typically, a material is a source control repository (like Git, Subversion, etc), and new commits to the repo will cause the pipeline to trigger. Pipelines require one or more material to be created.

    The concept of a pipeline is extremely central to Continuous Delivery. Together with the concepts of stages, jobs and tasks, GoCD provides important modeling blocks which allow you to build up very complex workflows, and get feedback quicker.
